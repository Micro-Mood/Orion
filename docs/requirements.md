# Personal AI — 需求文档

> 基于 Axon MCP 的个人 AI 助手

## 1. 项目定位

一个**类 Copilot 的个人 AI 助手**。

Copilot 是写代码的，Personal AI 是过日子的。设计理念完全一样：

- **自然对话**，不是命令行
- **AI 自己决定**什么时候用工具、怎么用，不需要人为规定流程
- **工具是手段**，对话才是核心
- **有记忆**，AI 自己管理，不需要用户操心存在哪

**一句话描述**：你的生活版 Copilot — 一个能聊、能做事、能记住的私人 AI。

### 架构概览

```
用户 ←→ 聊天前端（Web）
             ↕ WebSocket / HTTP
        后端 API Server（FastAPI）
           ↕              ↕
      LLM API（百炼）   Axon MCP Server（TCP:9100）
      模型FIFO降级        28个工具 · 文件/搜索/命令/系统
                          5层中间件 · 安全/验证/限流/并发/审计
```

### Copilot vs Personal AI

| | Copilot | Personal AI |
|---|---------|------------|
| 场景 | 写代码 | 过日子 |
| 工具 | 读代码、写代码、搜索符号、跑终端 | 读文件、写文件、搜索内容、跑命令 |
| 记忆 | 工作区文件就是上下文 | 工作区文件就是记忆 |
| AI 决策 | 自己判断该读文件还是直接回答 | 同理 |
| 用户体验 | 聊天框，流式输出，工具过程可见 | 同理 |

本质是同一个东西，只是 System Prompt 不同。

### 使用场景

```
用户：今天买了个键盘300块
AI：好的，已记下。

用户：我今年外设花了多少？
AI：（自动搜索 → 读文件 → 汇总）键盘300 + 鼠标150 = 450元。

用户：帮我解释一下什么是 HTTPS
AI：HTTPS 是...（直接回答，不调工具）

用户：看看服务器磁盘还剩多少
AI：（调 run_command → df -h）还剩 12GB。

用户：把我昨天的笔记整理一下
AI：（read_file → 整理 → write_file）已整理好。
```

## 2. 核心能力

### 2.1 对话
自然聊天，AI 有判断力。简单问题直接答，需要操作时自动调工具。

### 2.2 工具（Axon MCP）
通过 TCP 连接 Axon MCP Server，28 个工具随时可调。AI 自己决定用什么、怎么用。

### 2.3 记忆
文件就是记忆。AI 通过工具读写文件来记住和回忆。不预设目录结构 — AI 自己组织，就像 Copilot 不会规定你的代码目录结构一样。

### 2.4 上下文
FIFO 滑动窗口，长对话自动裁剪，Token 可控。

### 2.5 自驱动执行（参考 AutomateX）

和 AutomateX 一样的自驱动多轮循环，不是简单的一问一答：

```
用户输入
  ↓
AI 处理（自驱动多轮循环）
  ├─ 简单问题 → 直接回答 → done
  ├─ 需要操作 → 选择工具 → 执行 → 看结果 → 决定下一步
  │   ├─ 可能连续调用多个工具（搜索 → 读文件 → 写文件）
  │   └─ 中途需要用户确认 → ask → 等待 → 继续
  └─ 直到完成 → done
  ↓
等待下一次用户输入
```

#### 控制指令（与 AutomateX 一致）

| 指令 | 说明 |
|------|------|
| `done` | AI 认为已完成，返回最终回复 |
| `fail` | 失败，返回原因 |
| `ask` | 需要用户输入，暂停等待回答 |

#### 和 AutomateX 的区别

| | AutomateX | Personal AI |
|---|---|---|
| 触发方式 | 创建任务 → 自动启动 | **用户发消息 → 触发一轮** |
| 会话生命周期 | 有终态（完成/失败） | **持续存在**，用户随时继续 |
| 一轮结束后 | 任务终结 | **等待下一条消息**，开始新一轮 |
| 多轮之间 | 不同任务互相独立 | **同一会话共享上下文和历史** |

每次用户发消息都触发一轮自驱动循环。一轮结束（done）后，会话不关闭，等用户继续。

### 2.6 多会话
- 多个独立对话，侧边栏切换
- 每个会话独立的上下文和消息历史
- 参考 AutomateX 的 TaskStore 持久化

## 3. 技术方案

### 3.1 整体架构

| 层 | 组件 | 来源 | 说明 |
|----|------|------|------|
| 前端 | 聊天页面 | **新建** | 简洁对话 UI，流式输出，Markdown 渲染 |
| API 层 | FastAPI + WebSocket | 参考 AutomateX | HTTP + 流式推送 |
| LLM 层 | OpenAI 兼容 API | 参考 AutomateX | 百炼 API + FIFO 降级 |
| 上下文 | FIFO 滑动窗口 | 参考 AutomateX | Token 可控 |
| 工具层 | **Axon MCP Server** | ✅ **直接使用** | 28 个工具，独立进程 |

### 3.2 Axon MCP — 工具引擎

独立进程，JSON-RPC 2.0，TCP 127.0.0.1:9100。

**文件操作（14）**：`read_file` `write_file` `create_file` `delete_file` `stat_path` `list_directory` `move_file` `copy_file` `move_directory` `create_directory` `delete_directory` `replace_range` `insert_text` `delete_range`

**搜索（3）**：`search_text`（全文/正则） `find_files`（glob模式） `find_symbol`（代码符号）

**命令执行（10）**：`run_command` `create_task` `stop_task` `del_task` `task_status` `wait_task` `list_tasks` `read_stdout` `read_stderr` `write_stdin`

**系统（1）**：`get_system_info`

**内置安全（5 层中间件）**：
```
Security → Validation → RateLimit → Concurrency → Audit
路径黑名单    参数校验     写操作60/min   文件读写锁    操作日志
命令黑名单    类型检查     读操作300/min  防死锁排序    慢操作告警
```

### 3.3 模型 FIFO 降级

```
请求 → 模型1 → 成功 → 返回
              → 失败 → 模型2 → 成功 → 返回
                              → 失败 → 模型3 → ...
```

| 优先级 | 模型 | 输入/输出价格（每百万Token） | 说明 |
|--------|------|---------------------------|------|
| 1 | qwen-flash | 0.15 / 1.5 元 | 最便宜，日常够用 |
| 2 | qwen-turbo | 0.3 / 0.6 元 | 备选 |
| 3 | qwen-plus | 0.8 / 2 元 | 兜底 |

每个模型 100万 Token 免费额度（90天）。降级触发：API 报错 / 超时 / 额度用尽。

### 3.4 System Prompt

核心就是告诉 AI 两件事：
1. **你是谁** — 用户的私人生活助手，不是代码助手
2. **你有什么** — 一个工作区目录，你可以在里面自由读写文件来记住任何事情

其他的（什么时候记、怎么组织文件、怎么搜索）由 AI 自己判断，就像 Copilot 不需要被教怎么读代码一样。

### 3.5 上下文 FIFO

参考 AutomateX 的 `Context` 类：
- 滑动窗口（如 max_history=20）
- 自动裁剪最早消息
- System Prompt 始终保留

### 关键交互

| 元素 | 说明 |
|------|------|
| 侧边栏 | 会话列表，新建/切换/删除 |
| 聊天气泡 | 用户消息 + AI 消息，Markdown 渲染 |
| 工具调用 | 折叠在 AI 消息内，展开可看详情（工具名 + 参数 + 耗时 + 状态） |
| 流式输出 | WebSocket 推送 AI 思考过程 + 工具执行过程 |
| ask 状态 | AI 请求输入时，输入框高亮提示 |
| 移动端 | 响应式，侧边栏可折叠 |
| 部署 | 服务器子路径（如 domain/ai） |
